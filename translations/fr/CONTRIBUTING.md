# Guide de Contribution

Merci de votre intérêt pour le projet **Cybernetic Your Agent**. L'objectif de ce projet est de permettre à quiconque utilisant des Agents IA modernes de déployer facilement et de bénéficier des idées de la théorie de l'ingénierie cybernétique de Qian Xuesen.

Les contributions sous toutes formes sont les bienvenues.

---

## Directions de Contribution

Voici comment vous pouvez contribuer à ce projet :

### Ajout de Cas Pratiques

Déployez cette architecture sur différentes plateformes d'Agent et partagez votre expérience :
- Méthodes d'adaptation spécifiques à la plateforme (syntaxe d'appels d'outils, différences de mécanismes de mémoire, etc.)
- Problèmes rencontrés en pratique et leurs solutions
- Retours d'effet après une période d'utilisation (taux de corrections répétées, taux de réussite de préalable, etc.)

### Extension Théorique

Introduisez de nouveaux cadres théoriques pour étendre la couche de métacognition L1 :
- Par exemple : inférence bayésienne, théorie de l'information de Shannon, TRIZ, théorie des contraintes, etc.
- Requis : expliquer comment la nouvelle théorie travaille en synergie avec les quatre concepts existants, et fournir des éléments d'extension L1 exécutables

### Traduction

Traduisez le tutoriel dans d'autres langues. Veuillez d'abord ouvrir une Issue pour déclarer la langue que vous traduisez, afin d'éviter les doublons.

### Amélioration de Documentation

- Correction des fautes de frappe, erreurs grammaticales, formulations obscures
- Ajout de diagrammes (diagrammes d'architecture, diagrammes de flux de données)
- Amélioration des fichiers d'exemples

### Code et Scripts

- Fournir des scripts de déploiement automatique pour des plateformes d'Agent spécifiques
- Fournir des outils d'exportation d'état de mémoire/compétences

## Normes de Soumission

### Issue

- Avant de soumettre un bug, une suggestion d'amélioration ou une nouvelle idée, veuillez d'abord rechercher parmi les Issues existantes.
- Décrivez clairement le problème ou la suggestion, en incluant le contexte pertinent.

### Pull Request

1. **Fork ce dépôt**.
2. **Créez une branche distincte** pour la nouvelle fonctionnalité ou le correctif (par exemple `feat/add-claude-mcp-guide`).
3. **Gardez les messages de commit concis et clairs** (par exemple `docs: correction de l'explication du seuil de contrôle intégral du chapitre 3`).
4. **Associez l'Issue correspondante** (s'il y en a une), en utilisant `Closes #numéro_issue` dans la description de la PR.
5. Pour du nouveau contenu, **veuillez suivre le ton et la structure de la documentation existante** :
   - Concis, direct, orienté ingénierie
   - À toute théorie doit correspondre une mise en œuvre, à tout design doit correspondre une instruction
6. **Assurez-vous que le contenu est lisible de manière autonome** : chaque fichier de documentation doit être auto-contenu, sans dépendre du contexte de la conversation.

---

## Principes de Contenu

Ce tutoriel adhère aux principes suivants, veuillez les respecter lors de vos contributions :

- **L'exécutable prime sur la complétude théorique** : le contenu fourni doit être immédiatement opérable par le lecteur.
- **L'architecture prime sur l'accumulation** : nous ajoutons de la structure, pas des entrées de prompt.
- **Les fins ouvertes sont permises** : nous n'exigeons pas que chaque problème ait une solution ultime, mais donnons des directions d'investigation.

---

## Code de Conduite

- Les discussions se concentrent sur le tutoriel lui-même, en maintenant le respect et le caractère constructif.
- Les discussions rationnelles d'opinions divergentes sont les bienvenues, les attaques personnelles ne le sont pas.
- Les mainteneurs se réservent le droit de fermer les Issues/PR qui s'écartent du sujet ou ne suivent pas les normes.
